# Netflix Artwork A/B Testing (Experimentation Project)

## Overview
This project simulates a simplified version of Netflix’s artwork experimentation system. The goal is to understand how companies use A/B testing to make data-driven decisions instead of relying on intuition.

## Problem
When users browse Netflix, they make decisions quickly based on artwork (thumbnails). The challenge is:
Which image leads to higher engagement?

## Approach
We simulate an A/B test comparing:
- Version A (Artwork A)
- Version B (Artwork B)

Users are randomly assigned to each version, and we measure performance.

## Metrics
- Click-Through Rate (CTR)
- User engagement behavior
- Optional: watch time / completion rate

## Method
1. Randomly assign users to A or B
2. Track clicks and interactions
3. Calculate performance metrics
4. Compare results
5. Select the better-performing version

## Key Concept: Prediction vs Decision
Machine learning models can predict what might work, but experimentation shows what actually works.

This project demonstrates how A/B testing closes that gap.

## AI Factory Connection
- Data: user impressions and clicks
- Model: comparison logic between variants
- Prediction: which artwork may perform better
- Decision: which version to deploy
- Value: improved engagement

## Key Takeaways
- Small design changes can significantly impact user behavior
- Experimentation provides causal insights, not just predictions
- Data-driven decisions outperform intuition at scale
