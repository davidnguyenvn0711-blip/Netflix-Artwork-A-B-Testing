01 · Netflix Experiment Overview

The Setup

1,000Simulated Users
2Artwork Versions
50/50Random Split
24%Baseline CTR · A
30%Target CTR · B
42Random Seed

Artwork A is the control — the current default thumbnail. Artwork B is the challenger. 1,000 users were randomly split 50/50. Each user received one artwork version and their click decision was recorded. The simulation uses a fixed seed (42) so every run produces identical, reproducible output.

The experiment was run in three sequential steps: simulate user assignments and clicks → measure click-through rates per group → run a two-proportion z-test to confirm statistical significance.

03 · Netflix Business Impact

What This Means at Scale

Netflix serves 300+ million subscribers globally. A 25% relative improvement in click-through rate — sustained across even a fraction of that base — produces compounding downstream effects across revenue, operations, and supply chain.

From a supply chain lens, higher CTR is a leading demand signal. Content with strong early click-through warrants increased licensing investment, localization spend, and promotional budget allocation.

Content libraries are inventory. A/B testing enables data-driven decisions on which titles to promote, renew, or retire — reducing waste and improving catalog ROI, analogous to SKU rationalization in physical supply chains.

ML Feedback Loop

Each experiment's results become training data for Netflix's personalization engine — continuously improving demand forecasting accuracy for future content decisions. The model gets smarter with every test.

Supply Chain Parallel

In physical supply chains, demand planners run scenario analyses to test how small changes in product positioning, pricing, or packaging shift consumer purchase behavior. Netflix's A/B testing is the digital equivalent — controlled demand experiments at scale, where the "product" is content and the "shelf" is the homepage. The analytical discipline is identical: isolate one variable, measure the output, make the inventory decision.

A 25% lift in click-through in supply chain terms would be equivalent to a 25% uplift in sell-through rate after a packaging redesign — the kind of result that triggers immediate rollout, cross-functional review, and integration into demand forecasting models.

Step 1 · Simulate Users & Clicks
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import matplotlib.patches as mpatches
from scipy import stats

np.random.seed(42)

n_users = 1000
p_a     = 0.24  # Click probability — Artwork A (control)
p_b     = 0.30  # Click probability — Artwork B (challenger)

user_ids = np.arange(1, n_users + 1)
groups   = np.random.choice(['A', 'B'], size=n_users, p=[0.5, 0.5])

clicks = np.where(
    groups == 'A',
    np.random.binomial(1, p_a, size=n_users),
    np.random.binomial(1, p_b, size=n_users)
)

df = pd.DataFrame({
    'user_id':         user_ids,
    'artwork_version': groups,
    'clicked':         clicks
})

print(df.head(10))

Step 2 · Measure CTR & Visualize
summary = df.groupby('artwork_version')['clicked'].agg(
    total_users        = 'count',
    total_clicks       = 'sum',
    click_through_rate = 'mean'
).reset_index()

summary['click_through_rate'] = summary['click_through_rate'].round(4)

ctr_a = summary.loc[summary['artwork_version'] == 'A', 'click_through_rate'].values[0]
ctr_b = summary.loc[summary['artwork_version'] == 'B', 'click_through_rate'].values[0]
lift  = ((ctr_b - ctr_a) / ctr_a) * 100

print(f"CTR Artwork A : {ctr_a:.2%}")
print(f"CTR Artwork B : {ctr_b:.2%}")
print(f"Relative lift : {lift:+.1f}%")

fig, ax = plt.subplots(figsize=(8, 5))
colors   = ['#3a3a3a', '#E50914']
artworks = ['Artwork A', 'Artwork B']
bars     = ax.bar(artworks, [ctr_a, ctr_b], color=colors, width=0.45, zorder=3)

for bar, ctr in zip(bars, [ctr_a, ctr_b]):
    ax.text(bar.get_x() + bar.get_width() / 2,
           bar.get_height() + 0.003, f'{ctr:.2%}',
           ha='center', va='bottom', fontsize=13, fontweight='bold')

ax.annotate(f'Lift: {lift:+.1f}%', xy=(1, ctr_b),
    xytext=(1.15, ctr_b + 0.025), fontsize=10,
    arrowprops=dict(arrowstyle='->', lw=1.2))

ax.set_title('A/B Test: CTR by Artwork Version', fontsize=14, fontweight='bold', pad=14)
ax.yaxis.set_major_formatter(plt.FuncFormatter(lambda y, _: f'{y:.0%}'))
ax.grid(axis='y', linestyle='--', alpha=0.4, zorder=0)

for s in ['top', 'right']: ax.spines[s].set_visible(False)
plt.tight_layout()

Step 3 · Statistical Significance Test
clicks_a = df[df['artwork_version'] == 'A']['clicked'].sum()
clicks_b = df[df['artwork_version'] == 'B']['clicked'].sum()
n_a      = df[df['artwork_version'] == 'A'].shape[0]
n_b      = df[df['artwork_version'] == 'B'].shape[0]

p_pool  = (clicks_a + clicks_b) / (n_a + n_b)
se      = np.sqrt(p_pool * (1 - p_pool) * (1/n_a + 1/n_b))
z_stat  = (ctr_b - ctr_a) / se
p_value = 2 * (1 - stats.norm.cdf(abs(z_stat)))

print(f"Z-statistic : {z_stat:.4f}")
print(f"P-value     : {p_value:.4f}")
if p_value < 0.05:
    print("Result      : SIGNIFICANT — Artwork B wins. Ship it.")
else:
    print("Result      : Not significant — insufficient evidence.")
